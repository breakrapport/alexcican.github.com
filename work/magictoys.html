MagicToys

MagicToys is a company that sells physical magic products. I was hired to designed and develop a complete e-commerce solution that would act as a platform for selling and promoting physical as well as digital goods.

CMS Yes, Wordpress

Tools used Photoshop, Notepad, xHTML Strict 1.0, PHPurchase, CSS 3, jQuery

Concept Idea

The client requested that I design a full e-commerce solution for his business. Up until now, he used to sell magic products offline, by going to different fairs and presenting magic products to the audience around. He saw an opportunity of expanding his business online and decided to go with it. Moreover he wanted to sell digital products as well, such as video tutorials.

For the design, apart from wanting a commercial and simple layout, he left it entirely up to me to come up with design ideas. The final layout selected was a two column centre layout.

Some features of the website are: subscribe to the weekly newsletter, check for new products, add, edit and remove products from cart (visual feedback), search for a product, change quantity in cart, 1 step checkout, inventory checking, send e-mail confirmation on purchase, pay for products via “PayPal”, “Credit Card”, “Bank Transfer” or “Pay at Collection”, automatically generate receipts, sell and send digital products via e-mail, provide statistics of purchases, ability to add periodical discount codes, check if discount code is valid, contact MagicToys and more.

Production

I chose to use WordPress as the backbone of the site. By purchasing a plugin called PHPurchase, it would make handling of the products, invoices and automatic e-mailing, easier. I decided to go with PHPurchase (instead of other e-commerce plugins), because of its ability to customise the look of it completely. It was a bit of a pain to edit the plugin due to the many PHP files and complex coding but I finally pulled it off successfully.

The website was coded in valid xHTML Strict 1.0 and CSS 3. Once all the design elements were in place, it was then transposed into PHP and saved as a WordPress theme. I spent a significant amount of time on creating a nice visually pleasing menu in CSS3 that also works in IE (by adding images, screenshot on the right).


When adding a product to the cart, from an empty state (with empty cart icon) the cart was updated to a full state indicating the total price (excluding shipping) and the number of products. Once a product was added to the cart, a button appeared below leading you to the checkout page if clicked. Furthermore, because the cart was in the upper right corner, it could be easily missed, so when a product was added to the cart, a message bar slid down and notified you that the product was added successfully.

I wrote a tutorial on how to create that message bar. Check it out if you want: Display top message bar on click like Twitter →

The cart was designed with cookies, so the product would remain in the cart even after you would close the website (it would save the session until you closed the browser). This was a usability feature I wanted to include in this website and useful in case the Internet connection breaks down, or you close the tab of the browser accidentally.


The client had a marketing idea to keep customers coming back: When you purchased a product, a discount code was issued with the product giving you 10% – 40% off on your next purchase. Furthermore, he also wanted to have periodical discount codes, for example during Halloween. I provided a discount code checker that informed the user if the code they inserted was valid and how much discount they would get. If the code entered would not match the database records of active discount codes, it would display an error message.


The cart page and checkout page were completely redesigned to suit the needs of this magic product selling website. I tried making the process as easy and intuitive as possible. In the shopping cart page you can make changes to the cart and once you are done you click on the central big button “checkout”. That button, transfers you to the checkout page. From there, you can review your cart and select a payment method from the drop-down list.

In the contact form, if the fields completed were incorrect, using jQuery for error-checking, it would display errors with red and request the user to correct those fields. Moreover, checking the fields again in the PHP file is necessary, in case someone having JavaScript disabled tries to “trick” the system entering phony details.

In conclusion, this was a challenging project and I learned a lot in the e-commerce website design and development field. This made me gain confidence that I can develop from scratch, an impeccable fully working e-commerce website in a little over a month.
